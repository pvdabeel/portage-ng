# Reports

Regression analysis scripts and result data for portage-ng.

## Scripts

### `Reports/Scripts/compare-merge-emerge.py`

Compare portage-ng `.merge` output against Gentoo `.emerge` output for a
graph directory. Computes Jaccard-style accuracy metrics (CN, CN+V, CN+V+U),
ordering concordance, and assumption counts.

```bash
python3 -u Reports/Scripts/compare-merge-emerge.py \
  --root /Volumes/Storage/Graph/portage \
  --full-lists \
  --out Reports/compare-$(date +%Y-%m-%d)-$(git rev-parse --short HEAD).json
```

### `Reports/Scripts/compare-prover-failset.py`

Diff failed packages between two `prover:test(portage)` logs.

```bash
python3 Reports/Scripts/compare-prover-failset.py \
  --baseline baseline.log \
  --candidate candidate.log \
  --out Reports/prover_failset_compare.json
```

### `Reports/Scripts/compare-md5-cache.py`

Validate ebuild metadata against md5-cache entries.

### `Reports/Scripts/extract-timing.py`

Build a timing database from graph output files for build time estimation.

## Report data

JSON files named `compare-<YYYY-MM-DD>-<short-commit-hash>.json` are
generated by `compare-merge-emerge.py` and stored here for baseline
comparisons across commits.
